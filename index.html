    Problem Statement

    With over 70,000 games on Steam, players face decision paralysis when looking for their next game. Existing store recommendations rely on purchase history alone, missing opportunities to surface hidden gems based on gameplay preferences and genre affinity.

    Approach

    The system uses a hybrid recommendation approach. First, game metadata (genres, tags, descriptions) is vectorized using TF-IDF weighting. Cosine similarity between these vectors produces the content-based signal. Second, co-ownership patterns from SteamSpy data power collaborative filtering ‚Äî if users who own Game A disproportionately own Game B, that strengthens the recommendation.

    The final score combines both signals with tunable weights, and a diversity penalty prevents the results from clustering too heavily in one genre.

from sklearn.metrics.pairwise import cosine_similarity

tag_matrix = tfidf.fit_transform(games['combined_features'])

def get_recommendations(game_ids, n=10, content_weight=0.6):
    # Average the TF-IDF vectors for selected games
    user_profile = tag_matrix[game_ids].mean(axis=0)
    
    # Content-based scores via cosine similarity
    content_scores = cosine_similarity(user_profile, tag_matrix).flatten()
    
    # Collaborative filtering scores from co-ownership matrix
    collab_scores = get_collab_scores(game_ids, co_ownership_matrix)
    
    # Weighted hybrid with diversity re-ranking
    final = content_weight * content_scores + (1 - content_weight) * collab_scores
    return diversity_rerank(final, games, n)

    Results & Takeaways

    Evaluated with a held-out test set of 5,000 users: the hybrid model achieved a Hit@10 of 0.42, outperforming both the pure content-based (0.31) and pure collaborative (0.36) baselines. The diversity re-ranking step improved genre coverage by 28% without significantly impacting relevance scores.

    Key learning: blending content features with behavioral signals produces substantially better recommendations than either approach alone, especially for users with sparse play histories.
